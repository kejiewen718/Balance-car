# 项目分析

## 1. 项目本质

这是一个典型的两轮自平衡小车工程，核心是用 IMU 获取车体姿态，用编码器获取轮速，再通过三环控制输出到双路电机，让小车保持直立、前后运动和转向。

## 2. 当前工程的成熟度

从“能否作为参考工程”这个角度看，这个项目已经比较完整：

- 主控、驱动、传感器、控制器、显示和通信模块都具备
- 主控制闭环已经串通
- 有电压检测、参数保存、拿起/放下检测等安全逻辑
- 有蓝牙、超声波、红外、PS2 等扩展能力

但它仍然更像一套成熟教学工程，而不是完全精修的工业级代码库：

- 一些模块属于“可选扩展”，并没有在主流程中完全启用
- 个别实现偏经验式，优先考虑“够用、好调试”
- 部分代码存在历史遗留痕迹，例如注释和实际参数不总是完全一致

## 3. 主控制链路

### 高频闭环

`SysTick` 每 `5ms` 执行一次：

1. 读取陀螺仪和加速度计
2. 做加速度低通滤波
3. 进行姿态解算
4. 读取左右编码器
5. 计算角度环、速度环、转向环输出
6. 合成左右轮输出
7. 进行限幅、方向处理和 PWM 输出

### 低频任务

主循环中负责：

- 超声波触发测距
- 蓝牙控制包解析
- 电池电压检测
- OLED 参数显示

## 4. 控制框架

### 角度环

- 输入：当前倾角、角速度
- 作用：让车立起来
- 类型：PD

### 速度环

- 输入：左右轮编码器
- 作用：控制前进后退
- 类型：PI

### 转向环

- 输入：遥控目标、Z 轴角速度
- 作用：控制左右转向
- 类型：比例输出叠加陀螺反馈

## 5. 关键模块

### MPU6050

- 软件 I2C 通信
- 输出陀螺仪和加速度计原始数据
- 工程里包含零偏读取和校准存储逻辑

### 编码器

- 左右轮分别使用两个定时器的编码器模式
- 每个控制周期读取一次增量

### 电机驱动

- 两路 PWM 输出
- `TB6612` 提供方向与使能
- 输出前会做限幅和方向翻转

### Flash 参数存储

- 保存陀螺仪零偏
- 为 PID 参数保存留了结构

### 调试与交互

- 串口用于打印和上位机数据
- 蓝牙用于遥控
- OLED 用于本地观察关键状态

## 6. 适合作为你自己的基准仓库吗

适合。

最稳的使用方式不是直接在这份代码上大改，而是：

1. 保留当前仓库作为参考基线
2. 新建一个空白工程做自己的实现
3. 每完成一个阶段，就和这份工程对照
4. 有问题时回看这里的初始化顺序、采样周期和控制合成方式

这样即使你在新工程里写错了，也始终能回到这份参考仓库。
