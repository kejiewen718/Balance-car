# 从空白工程复现路线

## 目标

从一个全新的 `STM32F103C8` 工程开始，分阶段搭出一台可调试、可回退、可验证的自平衡小车，而不是一次性把所有模块堆进去。

## 推荐原则

- 原参考工程只读，不直接当实验场
- 每完成一个阶段就保存一次可运行版本
- 每一步都先验证输入和输出，再进入闭环
- 优先保证“方向正确”和“周期稳定”

## 阶段 1：最小系统

目标：

- 建立空白工程
- 跑通下载和复位
- 串口打印正常
- LED 能闪烁

验收标准：

- 上电后能看到固定周期串口输出
- LED 周期闪烁稳定

## 阶段 2：电机开环

目标：

- 配置 PWM
- 配置 `TB6612` 方向控制脚
- 让左右电机都能正转、反转、停止

验收标准：

- 左右轮可以独立控制
- 正负方向与你定义的车体前进方向一致

注意：

- 这一步就要把“左轮正方向”和“右轮正方向”统一好
- 如果方向定义错了，后面速度环和角度环都会变得很难调

## 阶段 3：编码器测速

目标：

- 配置编码器接口
- 周期读取左右轮增量
- 建立速度观测

验收标准：

- 手拨轮子时，左右轮数值变化方向符合预期
- 正转和反转时，编码器值符号正确

## 阶段 4：MPU6050 原始数据

目标：

- 跑通 I2C
- 读取加速度和陀螺仪原始值
- 确认静止时数据稳定

验收标准：

- 静止状态下陀螺仪零偏较小
- 翻动车体时三轴变化符合姿态变化

注意：

- 先不要急着上姿态解算
- 先确认安装方向、坐标轴和正方向定义

## 阶段 5：姿态解算

目标：

- 引入姿态解算
- 得到可用的车体倾角

验收标准：

- 车体前后倾时，倾角变化方向正确
- 静止时角度不明显漂移

注意：

- 你的 IMU 安装方向和参考工程不完全一样时，角度轴一定要重新对应
- 平衡车最关键的是“平衡角”和“对应角速度轴”配对正确

## 阶段 6：先调角度环

目标：

- 只启用角度环
- 让小车具备基本站立纠偏能力

验收标准：

- 手扶小车时，电机输出能朝纠正倾倒方向变化
- 不会出现一上电就明显反向暴冲

注意：

- 角度环未通之前，不要叠速度环
- 可以先悬空轮子观察输出方向

## 阶段 7：再加速度环

目标：

- 基于编码器引入速度环
- 让小车具备前后运动控制能力

验收标准：

- 小车静止时能逐渐收敛
- 给前进目标时能产生持续前进趋势

注意：

- 积分项必须限幅
- 倒地、停机、拿起时要清积分

## 阶段 8：最后加转向环

目标：

- 在不破坏平衡的前提下加入左右转向

验收标准：

- 左右转向时不会立刻失稳
- 转向停止后能回到稳定状态

## 阶段 9：做外设与交互

建议顺序：

1. 蓝牙遥控
2. OLED 显示
3. 电池电压检测
4. 超声波跟随 / 避障
5. 红外循迹
6. PS2 手柄

## 调参顺序

推荐固定顺序：

1. 角度环
2. 速度环
3. 转向环

不要反过来调，否则很难判断振荡来源。

## 最容易踩坑的地方

- IMU 姿态轴定义错
- 左右电机方向和编码器方向不统一
- 控制周期不稳定
- 电池压降导致供电扰动
- 电机噪声影响 IMU
- 没有在倒地或停机时清积分
- 超声波等阻塞逻辑侵入高频控制路径

## 版本管理建议

- 每个阶段单独提交一次
- 只有通过验收标准后再进入下一阶段
- 推荐的提交节奏：
  - `stage-1-minimum-system`
  - `stage-2-motor-open-loop`
  - `stage-3-encoder`
  - `stage-4-mpu6050-raw`
  - `stage-5-attitude`
  - `stage-6-balance-loop`
  - `stage-7-velocity-loop`
  - `stage-8-turn-loop`

## 最后建议

最稳的方法不是“把参考工程整份照抄”，而是“自己搭一遍，再和参考工程对照”。这样你后面才能真的改得动、调得动、还原得回去。
